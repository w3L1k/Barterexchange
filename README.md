# BarterExchange

BarterExchange — учебное web-приложение для обмена товарами. Проект сделан на Django и PostgreSQL, запускается через Docker Compose.

Главная идея приложения: пользователь публикует товар и указывает, что хочет получить взамен, а система подбирает подходящие объявления без привязки к цене или условной стоимости.

## Возможности

- регистрация, вход и выход по логину и паролю;
- демо-аккаунты для быстрой проверки;
- публичный каталог объявлений для гостей;
- создание, редактирование, архивирование и удаление объявлений владельцем;
- запрет редактирования, архивации и удаления объявления, если по нему есть активный обмен;
- фильтрация каталога по названию, категории, городу и состоянию;
- страница товара с описанием, владельцем и вариантами обмена;
- алгоритм подбора обменов по категории и городу;
- создание предложения обмена;
- входящие, исходящие, активные и завершенные обмены в профиле;
- статусы обмена: ожидает ответа, принят, отклонен, отменен, завершен;
- отзывы после завершенного обмена с оценкой от 1 до 5 звезд;
- профиль пользователя со списком объявлений и средней оценкой;
- уведомления о событиях обмена;
- жалобы на объявления и профили;
- модерация жалоб через Django admin;
- блокировка объявлений через админку.

## Логика подбора обменов

Подбор находится в `exchange/services.py`.

Система сравнивает объявление пользователя с другими активными объявлениями и сортирует варианты по приоритету:

1. Подходит желаемая категория и совпадает город.
2. Подходит желаемая категория, но город другой.
3. Совпадает город, но категория отличается.

Дополнительно учитывается обратный интерес: если владелец найденного объявления тоже ищет категорию товара текущего пользователя, такой вариант считается более удачным внутри своей группы.

Денежная ценность товара не используется. Это принципиальное решение проекта: обмен строится на интересе пользователей, а не на цене.

## Быстрый запуск

Нужны установленные Docker и Docker Compose.

```bash
docker compose up --build
```

После запуска сайт будет доступен по адресу:

```text
http://127.0.0.1:8000/
```

При первом запуске Docker Compose:

- поднимает PostgreSQL;
- применяет миграции;
- создает демо-данные, если база еще пустая;
- запускает Django dev server на порту `8000`.

SQLite в проекте не используется.

## Основные команды

Запустить проект:

```bash
docker compose up -d
```

Остановить контейнеры:

```bash
docker compose down
```

Остановить контейнеры и удалить данные PostgreSQL:

```bash
docker compose down -v
```

Посмотреть логи Django:

```bash
docker compose logs -f web
```

Применить миграции:

```bash
docker compose exec web python manage.py migrate
```

Создать новые миграции после изменения моделей:

```bash
docker compose exec web python manage.py makemigrations
```

Пересоздать демо-данные:

```bash
docker compose exec web python manage.py seed_demo
```

Запустить тесты:

```bash
docker compose exec web python manage.py test
```

Создать администратора:

```bash
docker compose exec web python manage.py createsuperuser
```

Админка:

```text
http://127.0.0.1:8000/admin/
```

## Модерация

Жалобы доступны в Django admin:

```text
Админка -> Обмен товарами -> Жалобы
```

Модератор может:

- посмотреть автора жалобы;
- посмотреть объект жалобы: объявление или профиль;
- изменить статус жалобы;
- указать результат рассмотрения;
- заблокировать связанные объявления.

Заблокированные и архивные объявления не показываются в каталоге и недоступны другим пользователям по прямой ссылке.

```text
BarterExchange
├── barterexchange/                 # настройки Django-проекта
│   ├── settings.py                  # конфигурация, PostgreSQL, static/media
│   ├── urls.py                      # корневые маршруты
│   ├── asgi.py
│   └── wsgi.py
├── exchange/                        # основное приложение
│   ├── models.py                    # модели: профили, объявления, обмены, отзывы, уведомления, жалобы
│   ├── views.py                     # обработчики страниц и действий
│   ├── forms.py                     # формы регистрации, объявлений, обменов, отзывов и жалоб
│   ├── services.py                  # алгоритм подбора обменов
│   ├── admin.py                     # настройка Django admin
│   ├── urls.py                      # маршруты приложения
│   ├── tests.py                     # автотесты
│   ├── migrations/                  # миграции базы данных
│   └── management/commands/
│       └── seed_demo.py             # наполнение базы демо-данными
├── templates/                       # HTML-шаблоны
│   ├── base.html                    # общий layout
│   └── exchange/                    # страницы приложения
├── static/                          # CSS, логотип, favicon, демо-картинки
│   └── exchange/
├── media/                           # загруженные пользователями изображения
├── docker-compose.yml               # web + PostgreSQL
├── Dockerfile                       # образ Django-приложения
├── requirements.txt                 # зависимости Python
├── manage.py
└── COMMANDS.txt                     # краткая локальная шпаргалка по командам
```

## Работа с изображениями

Демо-изображения лежат в:

```text
static/exchange/demo_images/stock/
```

Изображения, загруженные пользователями через форму создания или редактирования объявления, попадают в:

```text
media/listings/
```

Папка `media/` добавлена в `.gitignore`, потому что это пользовательские загрузки, а не исходный код проекта.

## Примечания

- Проект учебный и запускается с `DEBUG=True`.
- Для локальной разработки используется Docker Compose.
- Внешние сервисы авторизации не используются.
- Telegram Login, чат и расширенная система блокировки пользователей не входят в текущую версию.
