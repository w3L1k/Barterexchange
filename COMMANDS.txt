ШПАРГАЛКА ПО КОМАНДАМ ПРОЕКТА

Перейти в папку проекта:
cd "C:\Users\artyo\OneDrive\Документы\New project"

Запустить сайт через Docker:
docker compose up -d

Открыть сайт:
http://127.0.0.1:8000/

Остановить контейнеры:
docker compose down

Посмотреть запущенные контейнеры проекта:
docker compose ps

Посмотреть логи сайта:
docker compose logs -f web

Пересобрать контейнер после изменений в Dockerfile или requirements.txt:
docker compose up --build -d

Создать миграции после изменения models.py:
docker compose exec web python manage.py makemigrations

Применить миграции:
docker compose exec web python manage.py migrate

Заполнить базу демо-данными:
docker compose exec web python manage.py seed_demo

Демо-аккаунты после seed_demo:
логины: artem, maria, ilya, sofia, nikita, elena, daniil, alina
пароль у всех: demo12345

Запустить тесты:
docker compose exec web python manage.py test

Открыть Django shell:
docker compose exec web python manage.py shell

Создать администратора Django:
docker compose exec web python manage.py createsuperuser

Открыть админку Django:
http://127.0.0.1:8000/admin/

Где смотреть жалобы:
Админка -> Exchange -> Complaints

Что может модератор в жалобах:
- поменять статус жалобы: Новая, На рассмотрении, Рассмотрена, Отклонена
- написать результат рассмотрения в поле resolution
- массовым действием заблокировать связанные объявления

Очистить контейнеры проекта, но оставить данные PostgreSQL:
docker compose down

Очистить контейнеры проекта вместе с базой данных:
docker compose down -v

Важно:
После обычных изменений HTML, CSS или Python-кода контейнеры обычно пересоздавать не надо.
После изменения requirements.txt или Dockerfile нужна пересборка:
docker compose up --build -d
